Bad UI Battles

Build the most frustrating, impractical, and hilariously bad user form possible

Warning: May cause rage 10-15 minute challenge Maximum chaos

The Challenge

Your mission: Create a user registration form that works... technically... but is an absolute nightmare to use.

Think of every UX principle you've ever learned, then do the exact opposite. Make buttons that run away, inputs that fight back, and validation that makes no sense. The more ridiculous, the better!

Required Form Fields:

  • Name (but make it painful)
  • Email (but make it confusing)
  • Phone number (but make it absurd)
  • Password (but make it impossible)
  • Date of birth (but make it chaotic)
  • Terms acceptance (but make it evil)
  • Submit button (but make it elusive)

Terrible Ideas to Try:

  • Phone input using a rotary dial interface
  • Password that shows what you're typing to everyone
  • Submit button that moves when you hover
  • Date picker that only shows random dates
  • Checkboxes that are 3 pixels wide
  • Form that randomly resets itself
  • Validation messages in different languages
  • Required fields that aren't marked
  • Text inputs that type backwards
  • Success page that's just an error

Quick Code Examples

The Runaway Submit Button
button:hover {
    transform: translateX(200px);
    transition: all 0.3s;
}
The Helpful Password Field
<input type="text" placeholder="Password">
<div class="hint">

</div>
The Spinning Labels
label {
    animation: spin 4s linear infinite;
}
